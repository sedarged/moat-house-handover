## Runtime boundary

The web app is rendered inside the Windows WPF/WebView2 host in production.

Browser/Playwright execution is a test harness only. Do not turn the product into a browser-only app or add a production web-server requirement.

## UI rules

- Follow the existing modern Moat House dark/orange app direction.
- Do not copy old VBA/legacy visuals.
- Do not introduce a new UI framework unless explicitly requested.
- Reuse existing CSS patterns before adding new classes.
- Add reusable CSS classes when new styling is necessary.
- Keep screens dense, operational, and app-like.
- Avoid huge empty bottom gaps on operational screens.
- Use clear hover, focus, selected, and validation states.

## Data rendering and safety

- Do not interpolate user-entered or persisted data into `innerHTML` or HTML template strings.
- Prefer `textContent`, `input.value`, `textarea.value`, DOM creation, or explicit escaping helpers.
- Treat notes, reasons, comments, report paths, attachment names, department names, and user display values as unsafe unless proven static.
- Do not add new unsafe dynamic `innerHTML`.

## Host bridge and services

- Use the existing service/host bridge pattern.
- Do not bypass the service layer with hardcoded production data.
- Mock host bridge data belongs in tests/dev harness only.
- Keep payload names consistent between web services/screens and desktop-host contracts.

## Role/access rules

- Settings and Diagnostics are admin-only.
- Do not expose Settings/Diagnostics as normal operator/supervisor workflow actions.
- Normal users must not edit their Windows identity in the UI.

## Budget rules

Budget UI is a labour/budget workspace.

Do not mix it with efficiency, yield, or downtime metrics.

Required budget variance rule:

```text
Variance = Staff Used - Total Staff Required
```

Status mapping:

```text
variance > 0 = over
variance < 0 = under
variance == 0 = on target
```

Budget uses granular labour/budget areas and must not be forced into only the 13 handover departments.

## Verification for web/UI changes

For relevant UI or JavaScript changes, run:

```bash
npm run check:web
npm run test:ui
npm run screenshots
npm run verify:screenshots
```

If packaging can be affected, also run:

```bash
bash scripts/package-local.sh
bash scripts/verify-package-assets.sh
```

Screenshots generated by Playwright are browser/mock screenshots unless explicitly captured inside the real WPF/WebView2 host.

Do not claim Windows runtime verification from browser/mock screenshots.
